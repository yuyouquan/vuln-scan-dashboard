## 1. 产品概述

### 1.1 产品背景

随着企业安全意识的提升，漏洞扫描成为安全运营的重要环节。安全团队需要在 Bugcrowd 平台上管理多个扫描项目，实时跟踪扫描进度和结果。当前缺乏统一的看板工具来集中展示和管理这些扫描任务。

### 1.2 产品定位

一款面向安全团队的漏洞扫描项目管理看板，用于可视化展示 Bugcrowd 平台上的扫描项目状态、进度和详细信息。

### 1.3 核心价值

- **集中管理**：在一个页面查看所有扫描项目状态
- **实时监控**：实时展示扫描进度和运行状态
- **快速定位**：通过状态筛选快速找到目标项目
- **详情透明**：点击即可查看完整的扫描详情

## 2. 用户故事

### 2.1 安全工程师

> 作为一名安全工程师，我需要在一个页面查看所有 Bugcrowd 扫描项目的状态和进度，以便快速了解整体扫描情况，当有项目出现异常时能够及时发现和处理。

### 2.2 安全团队负责人

> 作为团队负责人，我需要查看团队负责的所有扫描项目概览，包括正在进行中、已完成、待处理的项目数量，以便汇报工作进度和资源调配。

### 2.3 安全运营人员

> 作为安全运营人员，我需要根据项目状态进行筛选，查看特定状态（如"扫描中"）的项目列表，并能够快速查看每个项目的详细扫描信息，包括使用的工具、扫描目标、发现的漏洞等。

## 3. 功能需求列表

### 3.1 需求概览

| 需求ID | 需求名称 | 优先级 | 描述 |
|--------|----------|--------|------|
| F-01 | 项目卡片展示 | P0 | 以卡片形式展示 Bugcrowd 扫描项目 |
| F-02 | 进度状态显示 | P0 | 显示每个项目的扫描进度和状态 |
| F-03 | 状态筛选功能 | P0 | 支持按状态搜索/筛选项目 |
| F-04 | 分页功能 | P0 | 支持分页，初始8条/页 |
| F-05 | 项目详情弹窗 | P0 | 点击卡片弹出 Modal 显示详细信息 |
| F-06 | 每页条数调整 | P1 | 支持切换 8/16/32 条每页 |

### 3.2 详细需求说明

#### F-01: 项目卡片展示

- 展示内容包括：
  - 项目名称（如 Under Armour Corporate, Tripadvisor）
  - 项目状态（待扫描、扫描中、已完成、已暂停）
  - 扫描进度（百分比 + 进度条）
  - 扫描目标数量
  - 创建时间
- 卡片布局：网格排列，响应式适配

#### F-02: 进度状态显示

- 状态类型：
  - `pending` - 待扫描
  - `running` - 扫描中
  - `completed` - 已完成
  - `paused` - 已暂停
  - `failed` - 失败
- 状态使用不同颜色标识：
  - 待扫描：灰色
  - 扫描中：蓝色
  - 已完成：绿色
  - 已暂停：黄色
  - 失败：红色
- 进度条动态展示扫描完成百分比

#### F-03: 状态筛选功能

- 搜索框：支持按项目名称模糊搜索
- 状态下拉筛选：支持多选状态进行过滤
- 筛选条件即时生效
- 支持清除筛选条件

#### F-04: 分页功能

- 默认每页显示 8 条记录
- 显示分页导航：页码 + 上一页/下一页
- 显示总记录数和当前页信息
- 支持跳转到指定页

#### F-05: 项目详情弹窗

- 点击卡片触发 Modal 弹窗
- Modal 内容包含：
  - 项目基本信息（名称、状态、创建时间）
  - 扫描目标列表（不同 API/地址）
  - 使用的扫描工具列表
  - 扫描内容描述
  - 运行状态/进度详情
  - 漏洞发现数量统计
- Modal 支持关闭（点击关闭按钮或遮罩层）

#### F-06: 每页条数调整

- 提供切换选项：8 条/页、16 条/页、32 条/页
- 切换后保持当前页码（若超出范围则回到第一页）
- 选项以Dropdown或Segmented Control形式展示

## 5. 交互流程

### 5.1 主流程

```
1. 用户打开看板页面
      ↓
2. 系统加载并展示项目卡片列表（默认第1页，每页8条）
      ↓
3. 用户可选择：
   A. [筛选流程] 进行状态/名称筛选 → 重新加载筛选结果
   B. [翻页流程] 切换分页查看更多项目
   C. [查看详情流程] 点击任意卡片 → 弹出详情Modal
   D. [调整显示] 切换每页条数 → 重新加载
```

### 5.2 筛选流程

```
1. 用户在搜索框输入项目名称
   或
   用户在状态下拉选择筛选条件
      ↓
2. 系统实时过滤项目列表
      ↓
3. 展示筛选后的结果，更新分页信息
      ↓
4. 用户可点击"清除筛选"重置
```

### 5.3 详情查看流程

```
1. 用户鼠标悬停在某个卡片上
      ↓
2. 卡片显示 hover 态（轻微放大/边框高亮）
      ↓
3. 用户点击卡片
      ↓
4. 系统弹出 Modal 框
      ↓
5. 用户阅读详情信息
      ↓
6. 用户点击关闭按钮或遮罩层
      ↓
7. Modal 关闭，返回看板页面
```

### 5.4 分页流程

```
1. 用户点击"下一页"或具体页码
      ↓
2. 系统加载对应页数据
      ↓
3. 页面滚动到顶部，展示新数据
      ↓
4. 分页组件更新当前页状态
```

## 6. 验收标准

### 6.1 功能验收

| 验收项 | 验收条件 |
|--------|----------|
| 项目卡片展示 | 页面加载后正确展示项目卡片，每个卡片包含名称、状态、进度条 |
| 状态颜色标识 | 不同状态显示对应颜色（灰/蓝/绿/黄/红） |
| 名称搜索 | 输入关键词后列表正确过滤，支持模糊匹配 |
| 状态下拉筛选 | 选择状态后列表正确过滤，支持多选 |
| 分页-初始8条 | 首次加载默认显示8条记录 |
| 分页-切换条数 | 选择16条或32条后列表正确更新 |
| 分页-翻页功能 | 点击页码正确切换数据 |
| Modal弹窗-触发 | 点击卡片后Modal正确弹出 |
| Modal弹窗-内容 | Modal内显示完整的项目详情信息 |
| Modal弹窗-关闭 | 点击关闭按钮或遮罩层Modal正确关闭 |

### 6.2 视觉验收

| 验收项 | 验收条件 |
|--------|----------|
| 卡片布局 | 卡片整齐排列，间距一致 |
| 响应式布局 | 桌面3列、平板2列、手机1列正确显示 |
| 进度条 | 进度条高度、颜色正确，百分比数字清晰可见 |
| 状态标签 | 状态文字与背景颜色对比度良好，易于识别 |
| Modal样式 | 弹窗居中显示，遮罩层半透明，关闭按钮可见 |

### 6.3 交互验收

| 验收项 | 验收条件 |
|--------|----------|
| Hover 效果 | 卡片 hover 时有视觉反馈 |
| 筛选响应 | 筛选操作后列表更新流畅 |
| 加载状态 | 数据加载时显示 loading 状态 |
| 空状态 | 无数据时显示友好提示 |

### 6.4 性能验收

| 验收项 | 验收条件 |
|--------|----------|
| 首屏加载 | 页面首次加载时间 < 2秒 |
| 筛选响应 | 筛选操作响应时间 < 500ms |
| 分页响应 | 切换分页响应时间 < 500ms |
